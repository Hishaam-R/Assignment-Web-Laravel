Barbershop
REST API

Laravel-powered REST API for the Ansar Barber appointment booking system. Supports public customer bookings, barber schedule management, and admin operations.

Framework Laravel (PHP)
Auth Method Session / Cookie
Database MariaDB
Format JSON
API Overview

The API is divided into three access tiers. Public endpoints require no authentication. Barber and Admin endpoints use session-based authentication (cookie) with role enforcement via the role:barber and role:super_admin middleware.

TierPrefixAuth RequiredRole
Public Booking/api/...No–
Barber Panel/api/barber/...Yesbarber
Admin Panel/api/admin/...Yessuper_admin

🧪 Testing with Postman

All endpoints can be tested with Postman or Advanced REST Client. For authenticated routes, you must first call the login endpoint, then include the session cookie in subsequent requests. In Postman, enable Cookies (or use the Cookie Manager) so the laravel_session cookie is sent automatically after login. Set header Accept: application/json and X-Requested-With: XMLHttpRequest on all requests to receive JSON error responses instead of HTML redirects.

Authentication Flow

Session-Based Authentication

This API uses Laravel's session authentication (not token-based). After a successful POST /api/barber/login or POST /api/admin/login, the server issues a session cookie. All subsequent authenticated requests must include this cookie. CSRF tokens are handled via the web middleware group.

Availability Rules
GET /api/barber/availability Get weekly schedule rules for the authenticated barber 🔒 ▶
Response 200 OK
{
  "data": [
    {
      "id": 1,
      "barber_id": 2,
      "day_of_week": 1,   // 0=Sun, 1=Mon, ..., 6=Sat
      "start_time": "09:00:00",
      "end_time": "17:00:00",
      "is_available": true
    }
  ]
}
POST /api/barber/availability Set/update a day's availability rule (upsert) 🔒 ▶
Request Body
{
  "day_of_week": 1,
  "start_time": "09:00",
  "end_time": "17:00",
  "is_available": true
}
Notes

Uses updateOrCreate — if a rule for that day_of_week already exists, it is updated. Otherwise a new rule is created. Returns 201 in both cases.
